# Integration Architecture Diagram Spec

## Purpose

Provide a diagram-ready specification for the integration model of the solution.

## Diagram Title

Integration Architecture: ESB-Led Connectivity Model

## Diagram Type

System integration diagram

## Core Components

### Internal Platform

- User interface layer
- Verification workflow services
- Reporting and audit services

### Integration Layer

- ESB / orchestration services
- Transformation services
- Routing and interface control
- Error handling and monitoring

### External Interfaces

- Department of Home Affairs
- Credit bureau services
- Banking validation services

### Internal / Government Interfaces

- Payroll / Persal-related services
- Departmental downstream systems

## Flows to Show

1. Verification request generated by the platform.
2. Request routed into the ESB.
3. ESB transforms and routes request to the required external interface.
4. Response returns through the ESB to the platform.
5. Platform stores results and updates reporting / audit services.

## Sequencing Notes

- DHA and credit bureau should be visually marked as high-priority integrations.
- Banking and payroll-related interfaces should be marked as staged or dependency-led integrations.
- GPT-approved IP and access controls should be called out on the DHA path.

## Required Annotations

- Centralized orchestration is used to reduce point-to-point complexity.
- Integration monitoring and auditability are required.
- Some interface details remain subject to discovery and customer-controlled readiness.
