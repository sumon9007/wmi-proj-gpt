# Activity Plan: Year 1

## Planning Basis

- Relative timeline only; no confirmed contract start date yet
- Year 1 covers design, build, integration, testing, training, deployment, and stabilization
- Activities assume gated dependencies for environment readiness and external integrations

## Month-by-Month Plan

| Timing | Focus Area | Key Activities | Key Outputs |
|---|---|---|---|
| Month 1 | Mobilization | Kickoff, governance setup, reporting cadence, RAID log initialization, stakeholder mapping | Initiation pack, governance calendar, initial RAID log |
| Month 2 | Discovery and baseline | Confirm scope, use cases, integration assumptions, security and compliance inputs, environment assumptions | Scope baseline, planning assumptions, dependency baseline |
| Month 3 | Detailed design | Produce functional design, architecture design, security plan, integration approach, reporting model | Design pack, security baseline, interface design baseline |
| Month 4 | Readiness gating | Confirm environment readiness, access pathways, DHA IP assumptions, hosting alignment, design approvals | Approved design baseline, environment-readiness review |
| Month 5 | Core build start | Build core platform services, user management, consent capture, workflow foundations, audit logging | Core build increment 1 |
| Month 6 | Core build continue | Extend verification workflows, reporting base, dashboard capability, exception handling, documentation updates | Core build increment 2 |
| Month 7 | Integration build | Begin DHA and credit bureau integrations, configure ESB/orchestration, validate core interface paths | Integration increment 1 |
| Month 8 | Integration expansion | Continue external integration work, begin banking and payroll-related onboarding where definitions permit | Integration increment 2 |
| Month 9 | Test readiness | Functional testing, integration testing, defect resolution, training material preparation, support model preparation | SIT readiness pack, draft training kit |
| Month 10 | Validation and training | Continue test cycles, validate resilience controls, finalize user and support training materials, operational documentation | Test evidence pack, training readiness |
| Month 11 | Deployment readiness | UAT / acceptance readiness, go-live planning, support handover preparation, production deployment planning | Deployment checklist, handover readiness pack |
| Month 12 | Go-live and stabilization | Production deployment, hypercare support, early-life issue resolution, acceptance closure | Go-live report, stabilization summary, acceptance record |

## Cross-Cutting Activities Throughout Year 1

- Weekly governance and status reporting
- RAID review and dependency escalation
- Security and compliance review checkpoints
- Documentation updates as build evolves
- Stakeholder engagement and decision tracking

## Gating Dependencies

- DHA connectivity through GPT-approved IP and access process
- SITA environment readiness and access provisioning
- Department of e-Government governance and security alignment
- Credit bureau service onboarding
- Banking and payroll-related interface definition

## Year 1 Exit Criteria

- Agreed functional scope delivered for go-live
- Required integrations enabled to the level approved for deployment
- Training and documentation completed for GPT IT first-line support
- Monitoring, backup, DR, and incident processes activated
- Acceptance and transition into months 13-36 support phase completed
