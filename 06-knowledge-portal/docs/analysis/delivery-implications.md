# Delivery Implications

## Delivery Model Implications

- the project should be managed as a two-phase engagement
- implementation and deployment must complete in months 1-12
- support and maintenance continue in months 13-36
- governance and dependency management must start early because several key inputs are customer-controlled

## Architecture Implications

- SITA-managed hosting is a non-negotiable design constraint
- an integration-centric architecture is required
- the ESB or orchestration layer should be explicit in the design
- capture, verification, integration, reporting, and audit functions should be separated for control and traceability

## Security And Hosting Implications

- security must be treated as a concrete design area, not a generic section
- resilience requirements such as uptime, backup, RTO, and RPO must be designed in from the start
- government ownership and control need to be reflected technically and contractually

## Implementation Implications

- prototype readiness should be treated as an early delivery stream
- DHA and credit bureau onboarding should be prioritized
- training and handover must include GPT IT first-line support preparation
- architecture, security, and implementation narratives need to stay tightly aligned to the requirement set
